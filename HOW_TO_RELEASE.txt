┌─────────────────────────────────────────────────────────────────┐

┌─────────────────────────────────────────────────────────────────┐

└─────────────────────────────────────────────────────────────────┘

│ BƯỚC 3: Đợi GitHub Actions build                               │
└─────────────────────────────────────────────────────────────────┘

1. Vào repository trên GitHub
2. Click tab "Actions"
3. Thấy workflow "Build SnapVideo" đang chạy (màu vàng)
4. Đợi 3-5 phút để build xong (màu xanh ✓)


┌─────────────────────────────────────────────────────────────────┐
│ BƯỚC 4: Kiểm tra Release                                       │
└─────────────────────────────────────────────────────────────────┘

1. Click tab "Releases" (hoặc code → Releases bên phải)
2. Thấy release "v1.0.0" mới được tạo!
3. File SnapVideo.exe đã được upload tự động
4. XONG! Có thể chia sẻ link release này!

📊 QUẢN LÝ VERSIONS
═══════════════════════════════════════════════════════════════════════

Quy tắc đặt tên version (Semantic Versioning):

v1.0.0 → vMAJOR.MINOR.PATCH

- MAJOR (1.x.x): Thay đổi lớn, không tương thích ngược
- MINOR (x.1.x): Thêm tính năng mới, tương thích ngược
- PATCH (x.x.1): Sửa lỗi, cải thiện nhỏ

Ví dụ:
- v1.0.0 → Release đầu tiên
- v1.0.1 → Sửa bug nhỏ
- v1.1.0 → Thêm tính năng mới
- v2.0.0 → Thay đổi lớn, giao diện mới

❓ FAQ
═══════════════════════════════════════════════════════════════════════

Q: Release và Artifacts khác nhau thế nào?
A: - Artifacts: Tự động, 90 ngày, cần login GitHub
   - Release: Public, vĩnh viễn, ai cũng download được

Q: Có thể xóa release?
A: Có! Vào Release → Edit → Delete release

Q: Có thể edit release sau khi publish?
A: Có! Click Edit release để sửa mô tả, thêm file

Q: Tag và Release khác gì?
A: Tag là marker trong Git, Release là GitHub feature
   Release = Tag + Mô tả + Files

Q: Có giới hạn số release?
A: Không! Unlimited releases

Q: Có giới hạn file size?
A: GitHub: 2GB/file, 10GB/release
   (SnapVideo.exe ~50MB → OK!)
